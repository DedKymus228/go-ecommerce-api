# Архитектура E-commerce API

**Стиль API:** RESTful  
**Базовый URL (Версионирование):** `/api/v1`

---

## 1. Авторизация и Пользователи (Auth)
*Публичные эндпоинты*

* `POST /api/v1/auth/register` — Регистрация нового пользователя.
* `POST /api/v1/auth/login` — Вход в систему (возвращает JWT токен).

---

## 2. Каталог товаров (Products)
*Публичные эндпоинты (Витрина)*

* `GET /api/v1/products` — Получить список всех товаров (поддерживает query-параметры для поиска и фильтрации, например: `?search=phone&category_id=2`).
* `GET /api/v1/products/:id` — Получить детальную информацию о конкретном товаре по его ID.

---

## 3. Корзина (Cart)
*Защищенные эндпоинты (Требуется JWT токен пользователя)*

* `GET /api/v1/cart` — Получить текущую корзину пользователя и список товаров в ней.
* `POST /api/v1/cart/items` — Добавить товар в корзину (в теле запроса передается `product_id` и `quantity`).
* `DELETE /api/v1/cart/items/:id` — Удалить товар из корзины.

---

## 4. Заказы и Оплата (Orders)
*Защищенные эндпоинты (Требуется JWT токен пользователя)*

* `POST /api/v1/orders` — Оформить заказ (Checkout). Корзина очищается, создается запись заказа со статусом "создан".
* `GET /api/v1/orders` — Получить список всех заказов текущего пользователя (история покупок).
* `GET /api/v1/orders/:id` — Получить детали конкретного заказа.
* `POST /api/v1/orders/:id/pay` — Имитация оплаты заказа (изменяет статус заказа на "оплачен").

---

## 5. Панель Администратора (Admin)
*Защищенные эндпоинты (Требуется JWT токен и роль Admin)*

* `POST /api/v1/admin/products` — Создать новый товар.
* `PUT /api/v1/admin/products/:id` — Обновить информацию о существующем товаре.
* `DELETE /api/v1/admin/products/:id` — Удалить товар из базы данных.
